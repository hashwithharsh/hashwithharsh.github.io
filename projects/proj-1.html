What is a Repository?

A repository (repo) is a software source from which Linux installs packages. Without a configured repo, package installation fails entirely.

bash
# Show all installed repos in system
dnf repolist

# Install a package (downloads from repo)
dnf install nginx

Types of Repositories

🟢 Online Repo

Internet-based, requires subscription. Used in dev/test/cloud systems.

🟢 Offline Repo

Uses ISO file, no internet required. Preferred in secure or air-gapped production environments.

Why DevOps Needs Repo Knowledge

ReasonWhy Important
StabilityPrevent unexpected updates
SecurityAvoid downloading from internet
ComplianceUse only approved packages
AutomationCI/CD needs controlled installs
TroubleshootingFix broken package systems
Production = Control + Predictability.

Offline Repo Setup

VM Method

bash
lsblk
sudo mount /dev/sr0 /mnt/iso

Dual Boot Method

bash
# -o loop: treats ISO file as virtual disk
sudo mount -o loop ~/iso/rhel-10.iso /mnt/iso

Create Repo File

Repo files live in /etc/yum.repos.d/

bash
sudo vi /etc/yum.repos.d/offline.repo
ini
[Offline-BaseOS]
name=Offline BaseOS
baseurl=file:///mnt/iso/BaseOS
enabled=1
gpgcheck=0

[Offline-AppStream]
name=Offline AppStream
baseurl=file:///mnt/iso/AppStream
enabled=1
gpgcheck=0
⚠ Must use file:/// — triple slash means file protocol with no hostname, rooted at the local filesystem.
bash
# Remove old cached metadata
sudo dnf clean all
# Download fresh repository metadata
sudo dnf makecache

Make Mount Permanent (fstab)

bash
sudo vi /etc/fstab
fstab
# Without nofail — required core mounts (/, /boot, /var, /home)
/home/harsh/iso/rhel-10.iso  /mnt/iso  iso9660  loop,defaults  0  0

# With nofail — non-essential mounts (NTFS drives, USB, ISO)
/home/harsh/iso/rhel-10.iso  /mnt/iso  iso9660  loop,defaults,nofail  0  0
💡 nofail allows the system to boot even if this mount fails. Only use on non-essential mounts. Core Linux mounts (/, /boot, /var) must never have nofail.
bash
# Test after every new fstab entry — no error = all good
sudo mount -a

Repo Control — Enable / Disable

Force Install from Specific Repo (Single Command)

bash
# Temporary — only for this single command
sudo dnf install tree --disablerepo="*" --enablerepo="Offline-*"

Permanent Enable / Disable

bash
# Disable a repo permanently
sudo dnf config-manager --set-disabled rhel-10-for-x86_64-appstream-rpms

# Enable a repo permanently
sudo dnf config-manager --set-enabled rhel-10-for-x86_64-appstream-rpms
Internally, these commands edit enabled=1 / enabled=0 inside the repo files at /etc/yum.repos.d/.

Repo Priority

bash
# Install priority plugin first
sudo dnf install dnf-plugins-core
ini
[rhel-10-for-x86_64-appstream-rpms]
name = Red Hat Enterprise Linux 10 AppStream
baseurl = https://cdn.redhat.com/...
enabled = 1
gpgcheck = 1
priority = 1  # Lower number = Higher priority

Version Locking

bash
# Install plugin
sudo dnf install python3-dnf-plugin-versionlock

# Check installed version before locking
dnf list installed nginx

# Lock current version
sudo dnf versionlock add nginx

# Lock specific version
sudo dnf versionlock add nginx-1.24.0-1.el10.*

# Lock major version only (allows minor updates)
sudo dnf versionlock add nginx-1.24.*

# List all locks
dnf versionlock list

# Remove a lock
sudo dnf versionlock delete nginx

# Remove all locks
sudo dnf versionlock clear
Locks are stored at /etc/dnf/plugins/versionlock.list — view with cat /etc/dnf/plugins/versionlock.list

Internal Repo Server (Enterprise)

In banks and secure enterprises, a single repo server serves all client machines. Architecture:

Repo Server → All Client Servers

bash
# 1. Install web server
sudo dnf install httpd

# 2. Copy ISO content
cp -r /mnt/iso/* /var/www/html/rhel10/

# 3. Allow firewall
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload

# Client repo file baseurl:
# baseurl = http://repo-server-ip/rhel10/BaseOS
