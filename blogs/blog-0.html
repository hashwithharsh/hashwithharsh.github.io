What is a Repository?

A repository (repo) is a software source from which Linux installs packages. Without a configured repo, package installation is impossible — no internet connection alone is enough.

bash
# Show all installed repos in system
dnf repolist

# Install a package (requires a configured repo)
dnf install nginx
Core Rule: Without a repo configured → ❌ No package installation. This is fundamental to RHEL system administration.

Types of Repositories

🟢 Online Repo

Internet-based repositories that require a Red Hat subscription. Typically used in dev, test, and cloud environments where internet access is available.

🟢 Offline Repo

Uses a local ISO file — no internet required. The go-to solution for secure, air-gapped production environments where external connectivity is restricted.

Offline Repo Setup

VM Method (Physical DVD or CD-ROM)

bash
lsblk
sudo mount /dev/sr0 /mnt/iso

Dual Boot / Loop Mount (ISO file on disk)

bash
sudo mount -o loop ~/iso/rhel-10.iso /mnt/iso

The -o loop flag makes Linux treat the ISO file as a virtual disk device — essential for mounting image files without physical media.

Creating the Repo File

All repo configurations live in /etc/yum.repos.d/. Create an offline.repo file pointing to your mounted ISO:

bash
sudo vi /etc/yum.repos.d/offline.repo
ini
[Offline-BaseOS]
name=Offline BaseOS
baseurl=file:///mnt/iso/BaseOS
enabled=1
gpgcheck=0

[Offline-AppStream]
name=Offline AppStream
baseurl=file:///mnt/iso/AppStream
enabled=1
gpgcheck=0
⚠ Critical: Always use file:/// (triple slash). The file protocol requires no hostname, so the third slash represents the root / of the local filesystem. Missing a slash is one of the most common configuration errors.

Clean & Rebuild Cache

bash
# Remove old cached metadata to prevent stale data issues
sudo dnf clean all

# Download fresh repo metadata for fast package resolution
sudo dnf makecache

Making the Mount Permanent (fstab)

Temporary mounts are lost on reboot. To persist the ISO mount, add it to /etc/fstab:

bash
sudo vi /etc/fstab
fstab
# With nofail — safe for non-critical mounts
/home/harsh/iso/rhel-10.iso  /mnt/iso  iso9660  loop,defaults,nofail  0  0
nofail rule: Add nofail to any mount not required for core Linux to boot (home dirs, extra ISOs, data drives). Never add nofail to critical system mounts like /, /var, /swap, or /boot/efi — those must succeed or the system won't boot correctly.
bash
# Always test fstab after every new entry!
sudo mount -a
# No output = success. Any error = fix your fstab entry before rebooting.

Force Install from Specific Repo

In secure or CI/CD environments, you may need to force package installation from a specific repo only (temporary, single-command scope):

bash
sudo dnf install tree --disablerepo="*" --enablerepo="Offline-*"

This disables all repos and enables only the offline ones — for that single command only. Normal repo settings are unchanged afterward.

Enable / Disable Repos

Temporary (per command)

bash
dnf install nginx --disablerepo="epel"

Permanent (edits enabled= in repo file)

bash
# Disable a repo permanently
sudo dnf config-manager --set-disabled rhel-10-for-x86_64-appstream-rpms

# Re-enable it
sudo dnf config-manager --set-enabled rhel-10-for-x86_64-appstream-rpms
Under the hood: These commands edit enabled=1 or enabled=0 in the repo files stored in /etc/yum.repos.d/. You can also edit them manually with vi.

Repo Priority

When multiple repos provide the same package, priority controls which one wins. Lower number = higher priority.

bash
# Install the priority plugin
sudo dnf install dnf-plugins-core
ini
[rhel-10-for-x86_64-appstream-rpms]
name = Red Hat Enterprise Linux 10 for x86_64 - AppStream (RPMs)
baseurl = https://cdn.redhat.com/...
enabled = 1
gpgcheck = 1
priority=1  # ← add this — lower = higher priority

Version Locking

Version locking prevents DNF from accidentally upgrading a package to a version that could break production. Critical for stable CI/CD environments.

bash
# Install the versionlock plugin
sudo dnf install python3-dnf-plugin-versionlock

# Always check current version before locking
dnf list installed nginx

# Lock current installed version
sudo dnf versionlock add nginx

# Lock a specific version manually
sudo dnf versionlock add nginx-1.24.0-1.el10.*

# Allow minor updates within 1.24, block 1.25+
sudo dnf versionlock add nginx-1.24.*
bash
# Check locked packages
dnf versionlock list

# Remove a specific lock
sudo dnf versionlock delete nginx

# Remove all locks
sudo dnf versionlock clear
Lock file location: /etc/dnf/plugins/versionlock.list — you can view this directly with cat /etc/dnf/plugins/versionlock.list.

Internal Repo Server (Enterprise Architecture)

In banks, enterprises, and high-security environments, a single internal repo server serves all client machines. No client ever touches the internet directly.

bash
# On the repo server
sudo dnf install httpd
sudo cp -r /mnt/iso/* /var/www/html/rhel10/
sudo firewall-cmd --add-service=http --permanent
sudo firewall-cmd --reload
sudo systemctl enable --now httpd
ini
# On all client servers — /etc/yum.repos.d/internal.repo
[Internal-BaseOS]
name=Internal BaseOS
baseurl=http://<repo-server-ip>/rhel10/BaseOS
enabled=1
gpgcheck=0
